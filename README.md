# Квиз-бот для Telegram

Бот квиз-сообщества: предстоящие игры по городам, рейтинг команд, FAQ, контакты.

## Структура проекта

```
quiz_bot/
├── bot.py                  # Точка входа
├── config.py                # Настройки (читает .env)
├── .env                      # Секреты (токен) — НЕ коммитить в Git!
├── .env.example              # Шаблон для .env
├── .gitignore
├── requirements.txt
├── handlers/
│   ├── start.py              # Главное меню
│   ├── games.py               # Предстоящие игры по городам
│   ├── rating.py               # Рейтинг команд
│   ├── faq.py                   # FAQ с иерархией
│   ├── contacts.py               # Контакты и соцсети
│   └── admin.py                   # Админ-команды
└── database/
    └── db.py                       # Работа с SQLite
```

## Установка локально

```bash
# 1. Клонировать репозиторий
git clone <ссылка-на-репозиторий>
cd quiz_bot

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Настроить .env
cp .env.example .env
# Открыть .env и вставить свой реальный токен от @BotFather

# 4. Запустить
python bot.py
```

## Деплой на сервер

1. Загрузить код на сервер (через `git clone` или `git pull`)
2. Создать файл `.env` на сервере **вручную** (он не приходит из Git — это специально, для безопасности)
3. Установить зависимости: `pip install -r requirements.txt`
4. Запустить бота как фоновый процесс, например через `systemd`, `screen`, `tmux` или Docker

### Пример запуска через screen (быстрый вариант)

```bash
screen -S quizbot
python bot.py
# Ctrl+A затем D — чтобы отключиться, оставив бота работающим
```

### Чтобы вернуться к запущенному боту:

```bash
screen -r quizbot
```

## Важно перед загрузкой в Git

Убедись, что в репозиторий **не попадает**:
- `.env` (содержит токен)
- `*.db` (база данных — на сервере создастся новая)

Оба этих правила уже прописаны в `.gitignore`.

## Админ-команды (только для ID из ADMIN_IDS)

```

/list_games
/del_game ID
/add_result Игра | Дата | Команда | Место | Баллы
/admin
```
